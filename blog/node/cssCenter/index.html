# 1、水平居中

a) inline-block + text-align










<style>
  .parent{
    text-align: center;
  }
  .child{
      display: inline-block;
  }
</style>

tips：此方案兼容性较好，可兼容至IE8，对于IE567并不支持inline-block，需要使用css hack进行兼容

b) table + margin








<style>
  .child{
    display: table;
    margin: 0 auto;
  } 
</style>

此方案兼容至IE8，可以使用table代替css写法，兼容性良好

c) absolute + transform














<style>
  .parent{
    position: relative;
    height:1.5em;
  }
  .child{
    position: absolute;
    left: 50%;
    transform: translateX(-50%);
  }

</style>

tips：此方案兼容至IE9，因为transform兼容性限制，如果.child为定宽元素，可以使用以下写法，兼容性极佳















<style>
.parent{
    position: relative;
    height:1.5em;
}
.child{
    position: absolute;
    width:100px;
    left: 50%;
    margin-left:-50px;
}

</style>

d) flex + justify-content












<style>
  .parent{
    display: flex;
    justify-content: center;
  }
  .child{
    margin: 0 auto;
  }

</style>

tips：flex是一个强大的css，生而为布局，它可以轻松的满足各种居中、对其、平分的布局要求，但由于现浏览器兼容性问题，此方案很少被使用，但是值得期待浏览器兼容性良好但那一天！

# 2、垂直

a) table-cell + vertial-align









<style>
  .parent{
    display: table-cell;
    vertical-align: middle;
  }

</style>

tips：可替换成table布局，兼容性良好

b) absolute + transform












<style>
  .parent{
    position: relative;
  }
.child{
	position: absolute;
	top: 50%;
	transform: translateY(-50%);
}
</style>

tips：存在css3兼容问题，定宽兼容性良好

c) flex + align-items









<style>
  .parent{
    display: flex;
    align-items: center;
  }

</style>

tips：高版本浏览器兼容，低版本不适用

# 3、水平垂直

1) inline-block + table-cell + text-align + vertical-align,父元素不能使用浮动，使用浮动垂直居中失效













<style>
  .parent{
    text-align: center;
    display: table-cell;
    vertical-align: middle;
  }
  .child{
    display: inline-block;
  }

</style>

tips：兼容至IE8

2) absolute + transform














<style>
  .parent{
    position: relative;
  }
  .child{
    position: absolute;
    left: 50%;
    top: 50%;
    transform: translate(-50%,-50%);
  }

</style>

tips：兼容性稍差，兼容IE10以上

3) flex最优雅的方法










<style>
  .parent{
    display: flex;
    justify-content: center;
    align-items: center;
  }

</style>

tips：兼容差

4)margin+transform











<style>
  .parent{
   overflow:hidden;
  }
  .child{
    margin:50% auto;
    transform:translateY(-50%)
  }
</style>

5)inline-block+vertical-align












<style>
  .parent{
   text-align:center;
   line-height:300px;
  }
  .child{
    display:inline-block;
    vertical-align:middle;
  }
</style>

6)abslute+负margin












  .parent{
    position: relative;
    }
  .child{
    position :absolute;
    left:50%;
    top:50%;
    margin-left:-50px;
    margin-top:-50px
  }

7)absoulte+margin:auto













  .parent{
    position:relative;
  }
  .child{
    position:absolute;
    left:0;
    right:0;
    top:0;
    bottom:0;
    margin:auto;
  }

8)最新的方法grid









  .parent{
    display:grid;
  }
  .child{
    align-self:center;
    justify-self:center;
  }

# 4、多列布局--一列定宽，一列自适应

a) float + margin












<style>
  .left{
    float: left;
    width: 100px;
  }
  .right{
    margin-left: 120px;
  }

</style>

tips：此方案对于定宽布局比较好，不定宽布局推荐方法b

b) float + overflow













<style>
  .left{
    float: left;
    width: 100px;
    margin-right: 20px;
  }
  .right{
    overflow: hidden;
  }

</style>

tips：个人常用写法，此方案不管是多列定宽或是不定宽，都可以完美实现，同时可以实现登高布局

c) table
















<style>
  .parent{
    display: table; width: 100%;
    table-layout: fixed;
  }
  .left,.right{
    display: table-cell;
  }
  .left{
    width: 100px;
    padding-right: 20px;
  }

</style>

d) flex














<style>
 .parent{
    display: flex;
  }
  .left{
    width: 100px;
    padding-right: 20px;
  }
  .right{
    flex: 1;
  }
</style>

# 16、绝对定位布局

优点：绝对定位布局，很容易，效率也很高

缺点：实际开发中很少使用，原因也很简单，绝对定位的元素是脱离文档流的，可维护性会受限 










<div class="JD">

  </div>
  <div class="JD_right"></div>

    width: 300px;
    background: red;
}

# 17、普通浮动布局

这种布局主要是控制元素浮动来实现的，要注意的一点就是中间元素要创建BFC，否则一旦高度变化就会无法正常工作。

优点：浮动对旧浏览器兼容性好；

缺点：主体内容需要放到最后加载，当页面元素较多时候可能会影响体验 










  <div class="container">
      <div class="left"></div>
      <div class="right"></div>
      <div class="center">
          <h1>浮动布局</h1>
      </div>
  </div>
























<style>
.container>div{
    height: 200px;
}
.left {
    float: left;
    width: 300px;
    background: red;
}
.right {
    float: right;
    width: 300px;
    background: blue;
}
.center {
  overflow: hidden;
  background: yellow;
}

</style>

# 18、flex布局

把外层容器显示属性设置成flex，里面只要自适应部分flex设置为1，就可以实现自适应效果了。

使用flex布局的代码特别简洁，也是实现自适应布局的最佳方案，唯一的问题就是旧浏览器不兼容这一布局方式。

现在需要适配旧浏览器的场景越来越少了，尤其是移动开发，flex可以完全放心使用。 










<div class="FX">
  <div class="FX_left"></div>
  <div class="FX_center">
      <h1>flexbox</h1>
  </div>
  <div class="FX_right"></div>
</div>

























<style>
.FX {
    display: flex;
}
.FX_container>div {
    height: 200px;
}
.FX_left {
    width: 300px;
    background: red;
}
.FX_center {
    flex: 1;
    background: yellow;
}
.FX_right {
    width: 300px;
    background: blue;
}

</style>

# 19、table布局

把外层容器设置成table,里面设置为table-cell，就可以很容易地实现布局需求。。

优点：兼容性还特别好，因为表格是兼容旧浏览器的，虽然遭受很多诟病，但是真的可以解决问题

缺点：就是不灵活，边框设置、高度设置等等都有很大受限 










<div class="table">
  <div class="table_left"></div>
  <div class="table_center">
      <h1>表格布局</h1>
  </div>
  <div class="table_right"></div>
</div>


























<style>
.table {
    width: 100%;
    display: table;
    height: 200px;
}
.table>div {
    display: table-cell;
}
.table_left {
    width: 300px;
    background: rgb(146, 124, 124);
}
.table_center {
    background: rgb(41, 39, 155);
}
.table_right {
    width: 300px;
    background: greenyellow;
}

</style>

# 20、网格布局

网格布局，这个布局是新的css标准下的特性,在响应式布局大行其道的移动互联网时代，bootstrap之类的是对栅格化布局框架非常流行，而网格布局，就是对栅格布局的标准化实现。










<div class="WG">
  <div class="WG_left"></div>
  <div class="WG_center">
      <h1>网格布局</h1>
  </div>
  <div class="WG_right"></div>
</div>






















<style>
.WG {
  display: grid;
  width: 100%;
  grid-template-rows: 200px;
  grid-template-columns: 300px auto 300px;
}
.WG_left {
  background: red;
}
.WG_center {
  background: yellow;
}
.WG_right {
  background: blue;
}

</style>
